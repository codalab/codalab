Competition Element Reference

This document describes the elements of a competition. Click an element in the example YAML file to see the definition for the element.

Example YAML

title: Example Competition
description: This is an example competition.
image: logo.jpg
has_registration: True
html: 
    overview: overview.html
    evaluation: evaluation.html
    terms: terms_and_conditions.html
    data: data.html
phases:
    1:
        phasenumber: 1
        label: "Test"
        start_date: 2013-06-30
        max_submissions: 100
        scoring_program: program.zip
        reference_data: reference.zip
        datasets: 
            1:
                name: Data 1
                url: http://www.example.com/data.xls
                description: Example Dataset
leaderboard:
    leaderboards:
        RESULTS: &RESULTS
            label: Results
            rank: 1
    columns:
        Difference:
            group: *RESULTS
            label: Difference
            rank: 1
            numeric_format: 6

Competition Elements

This section describes the elements that comprise a competition.

The actual scores in the leaderboard.

difference

A column that represents the internal key for this score.

end_date

The end date for the competition. It is valid to not include an end_date, which means the competition remains open perpetually.

back to top

evaluation

The HTML page that describes how the scores for this competition are evaluated.

The group to which this score belongs.

Indicates whether the competition requires participant approval by the organizer.

html

Defines the competition's HTML pages. Each competition has a set of pages that describe the competition and contain instructions for participants.

The filename of the logo or image for the competition.

label

The label (name) of the phase, group, or score.

leaderboard

Presents collections of scores, and enables scores to be grouped.

back to top

max_submissions

The maximum number of submissions per participant.

back to top

name

The name of the dataset.

back to top

numeric_format

The numeric format to use for the column.

back to top

overview

The HTML page that contains an overview of the competition.

back to top

phasenumber

The phase number for ordering.

phases

Competitions are broken up into phases. Each competition has at least one phase, and can have multiple phases. For example a competaition may have an evaluation phase as well as a competition phase.

back to top

rank

Rank order of a group or score within a list of groups or scores.

back to top

reference_data

A ZIP archive containing reference data to compare submitted data with for scoring.

back to top

results

The internal key name for the overall results group.

back to top

scoring_program

A ZIP archive containing the program that is used to evaluate results.

back to top

start_date

The start date for the phase.

back to top

terms

The HTML page that contains terms for participating in the competition.

back to top

The title of the competition.
